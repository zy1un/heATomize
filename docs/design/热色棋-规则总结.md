# 热色棋：规则总结

> 版本：v0.4，2026-03-30

---

## 一、基本设定

| 项目 | 设定 |
| ---- | ---- |
| 棋盘 | `9 x 9` 正方网格 |
| 邻域 | 4 邻域：上、下、左、右 |
| 球的唯一属性 | 热度 `1..5` |
| 视觉映射 | 低热度偏冷，高热度偏亮、偏热 |
| 结束条件 | 玩家无法移动任何球到任何可达空格时判负 |

---

## 二、每回合流程

1. 玩家移动一个球。移动路径必须沿空格走，且使用 4 邻域可达。
2. 系统随机生成 `min(3, 空格数)` 个新球，新球热度从 `1..3` 中随机。
3. 执行系统循环：
   - Phase 1：所有球根据旧棋盘状态同步更新热度。
   - Phase 2：检测同热度连通体，达到阈值则消除。
   - Phase 3：被消除格子的正交邻居发生余震，热度 `+1`，上限为 `5`。
4. 如果 Phase 2 产生了新的消除，则继续回到 Phase 1。
5. 没有新的消除后，检查是否 game over。

---

## 三、热度变化 Phase 1

每个球每轮最多变化 `1` 点，且所有球都用旧值同步计算。

| 条件 | 变化 |
| ---- | ---- |
| 存在任意邻居比自己更热 | 热度 `+1` |
| 所有邻居都比自己更冷，或没有邻居 | 热度 `-1` |
| 其余情况 | 不变 |

热度始终截断在 `[1, 5]`。

直觉：热源旁边的球会被烤热；孤立或周围全是冷球的热球会慢慢冷却；同温集群彼此稳定。

---

## 四、消除检测 Phase 2

系统搜索所有同热度的 4 邻域连通体。若连通体大小达到当前热度的阈值，则整组消除。

| 热度 | 阈值 | 含义 |
| ---- | ---- | ---- |
| 1 | 7 | 最稳定 |
| 2 | 6 | 稳定 |
| 3 | 5 | 中等 |
| 4 | 4 | 不稳定 |
| 5 | 3 | 极不稳定 |

核心逻辑：越热越不稳定，越少的球聚在一起也会爆。

---

## 五、余震 Phase 3

消除后，所有被消除格子的正交邻居如果仍有球，则热度 `+1`，上限为 `5`。

余震后重新进入 Phase 1 和 Phase 2，因此可能形成级联反应。

---

## 六、核心涌现特性

### 定时炸弹效应

小团体不会立刻爆，但随着热传播可能逐渐升温，最终在高热度阶段以更少数量触发消除。

| 集群大小 | 可能爆发热度 | 感觉 |
| ---- | ---- | ---- |
| 7 个及以上 | 1 | 原地就能爆 |
| 5 到 6 个 | 2 或 3 | 短引信 |
| 4 个 | 4 | 中期威胁 |
| 3 个 | 5 | 长引信定时炸弹 |
| 2 个 | 不会自爆 | 等待第三个或被余震推动 |

### 热波传播

余震制造热源，热源再推动外圈升温，外圈同温球逐渐达到阈值后再次消除，形成肉眼可见的链式热波。

### 路径障碍

移动需要 4 邻域可达路径。棋盘上的球群会天然形成障碍，玩家需要绕路、清障或借消除重塑路径。

---

## 七、玩家策略空间

| 策略 | 描述 |
| ---- | ---- |
| 主动引爆 | 将同温球移到一起触发消除，清出空间 |
| 引线布局 | 安排球的位置，引导余震传播方向 |
| 隔热防火 | 移开球切断传导路径，避免不想要的连锁 |
| 未爆弹管理 | 保留两个高热球，等待第三个到位引爆 |
| 路径博弈 | 利用障碍和空位管理移动路线 |

---

## 八、待确认参数

| 参数 | 当前值 | 备注 |
| ---- | ---- | ---- |
| 消除阈值 | `7/6/5/4/3` | 需要更多 playtest 验证 |
| 每回合生成球数 | `3` | 可根据节奏调整 |
| 新球热度范围 | `1..3` | 不直接生成高热球 |
| 余震强度 | 统一 `+1` | 可考虑高热消除产生更强余震 |

---

## 九、设计哲学

一条状态维度：热度。

一条温度规则：有更热的邻居就升温；比所有邻居都热就降温；否则稳定。

一条消除规则：同温连通体足够大就爆，且越热阈值越低。

一个传播机制：余震给邻居 `+1`。

这些规则共同涌现出热波扩散、定时炸弹、级联反应和路径管理。
