# 计分与打击感 (Game Feel) 实现计划

本计划将当前的计分规则转化为分阶段的实施路径，以实现更强的反馈、更清晰的得分展示以及更具冲击力的连锁反应。

## 设计解读

新的计分方向非常出色，因为它赋予了 `heATomzie` 独特的个性：得分不再仅仅是数字，它们是热量、电压和被吸入反应堆式计分板的原子碎片。

最稳妥的实现路径是循序渐进的。分层构建得分反馈，在视觉上逐层验证，并避免在节奏未调优之前添加过多同时发生的特效。

最重要的原则：普通移动应当保持清晰和（相对）安静；而连锁反应则应当逐步升级为视觉奇观。

## 核心特性

1. 辉光管 (Nixie) 风格总分板
2. 轻量级的生存分反馈
3. 消除分数弹出与原子化粒子流
4. 连击乘数横幅与顿帧 (Hit stop)
5. 屏幕震动与高能冲击特效
6. 连击动画加速
7. 无障碍 (Accessibility) 友好的特效强度设置

## 阶段 1：得分反馈基础

目标：使现有的计分系统可视化，同时不干扰核心玩法。

任务：
- 在 HUD 或 Main 场景中添加一个专用的得分反馈层。
- 从 `board.gd` 发射 (emit) 得分事件，而不是让 HUD 仅仅依据总分推断。
- 添加三种得分事件类型：
  - `survival` (生存)：每次成功移动后固定 `+2`。
  - `clear` (消除)：每个被消除的群组，单球得分 `10 + 2 * 热度`。
  - `multiplier` (乘数)：显示连击深度，如 `x2`, `x4`, `x8` 等。
- 保持生存反馈微小，并局限在移动目标位置附近。
- 保持消除分数在被消除格子的附近弹出。
- 保持乘数反馈在屏幕居中，并仅在连击深度 `>= 2` 时显示。

实施建议：
- 在 `board.gd` 中添加 `score_event` 信号。
- 创建一个新脚本，例如 `scripts/ui/score_feedback.gd`。
- 将得分动画代码与 `board.gd` 核心逻辑分离。

## 阶段 2：辉光管 (Nixie) 计分板

目标：让总分板感觉像一台物理热能设备。

任务：
- 将普通的分数文本标签替换为带有辉光管灵感的得分显示器。
- 使用飞速滚动的数字来制作分数上涨的动画（类似老虎机）。
- 分数注入时，短暂增加辉光效果。
- 为高乘数添加“过载 (overload)”状态。

首版建议：
- 先使用带样式的 `Label` 节点，暂不用自定义 Shader。
- 通过主题颜色和复制的带阴影标签来制作橙/黄色发光效果。
- 通过让数字快速跳过临时值再停在真实分数上，来实现滚动效果。

后续版本：
- 使用自定义数字纹理或发光 Shader。
- 在高乘数时，得分面板周围增加火花粒子。

## 阶段 3：原子化得分粒子

目标：使分数在物理上表现出从被消除的球中飞向计分板的过程。

任务：
- 消除发生时，在消除格子处生成对应颜色的分数数字。
- 数字短暂悬停、弹跳，随后破碎为正方形或点状的细小粒子。
- 粒子流向辉光管计分板飞去。
- 粒子的颜色必须与被消除球的热度颜色严格一致。

实施注意事项：
- 复用当前像素碎片爆炸的视觉语言。
- 初期控制粒子的生成数量，不宜过多。
- 对于大规模消除，尽量打包生成群组得分粒子，以避免单球生成过多文字标签性能下降。

## 阶段 4：连击乘数冲击

目标：让余震惊发（引发消除）的得分感觉截然不同并且充满重量感。

任务：
- 当 `current_chain >= 2` 时，在屏幕中央展示乘数横幅。
- 乘数出现时使用顿帧 (hit stop)。
- 将消除产生的得分粒子先吸入乘数横幅，再输送入计分板。
- 根据乘数大小按比例放大横幅尺寸和发光强度，并设置合理的上限。

推荐阈值：
- `x2`: 中等横幅，极短的顿帧。
- `x4`: 更大的横幅，更强的发光。
- `x8+`: 过载表现，屏幕震动，四周爆出火花。

## 阶段 5：顿帧与屏幕震动

目标：在不损害游戏可读性的前提下增加物理重量感。

任务：
- 添加一个可复用的顿帧辅助函数。
- 添加一个可复用的屏幕震动辅助函数。
- 消除热度 4 或 5 的集群时，触发短促顿帧。
- 发生余震热波传导时，触发微弱且急促的屏幕震动。
- 高倍数连击时，触发更强烈的震动。
- 增加 Game Over 时的停机顿帧及画面逐渐暗淡的谢幕序列。

实施警告：
- 目前请确保所有的震动都在游戏视口 (viewport) 内进行。
- 暂时不要在当前计划中实现操作系统的物理窗口震动；因为它依赖特定平台、且无法在 headless 环境下轻易测试，而且有可能会惹恼部分玩家。

## 阶段 6：连击加速

目标：初期的消除和传导保持极佳的可读性，随后不断的连击会越来越快，演变成一场视觉奇观。

设计：
- 第 1 波：正常速度 `1.0x`
- 第 2 波：约 `1.25x` 速度
- 第 3 波：约 `1.5x` 速度
- 第 4 波及以上：极限限制在 `2.0x` 到 `2.5x`

节奏乘数将作用于：
- 热量传导的粒子持续时间
- Phase 间的阶段暂停
- 消除收缩动画的时间
- 余震浪潮的传递动画时间
- 得分粒子的飞行时间

节奏乘数不作用于：
- 玩家的移动动画
- 初期生成新球的展示（需保持可读）
- 底层规则的计算

实施注意事项：
- 添加类似 `get_chain_tempo(chain_depth: int) -> float` 的获取函数。
- 动画耗时一律使用 `duration / tempo`。
- 等待时间一律使用 `pause / tempo`。
- 本地战术模式 (Tactical Mode) 加速上限可稍低。
- 混沌模式 (Chaos Mode) 可设置更高的极限速度。

## 阶段 7：视觉测试

目标：阻止花哨的特效意外遮挡或破坏游戏局势的可读性。

任务：
- 针对以下场景添加基于截图的冒烟测试 (smoke checks)：
  - 正常棋盘加载后状态
  - 规则面板可见性
  - 计分板可见性
  - 被选中的球可见性
  - 消除反馈场景
  - 超高乘数场景
- 提供一张确定性的“调试专用棋盘”，专门用于计分和连击反馈的自动化测试。

## 推荐执行顺序

1. 建立得分事件信号和独立的反馈层。
2. 实装生存分与基础消除分的弹窗。
3. 实装辉光管 (Nixie) 风格总分板。
4. 实装乘数横幅。
5. 加入顿帧与屏幕震动。
6. 加入连击加速。
7. 打磨原子化粒子流效果。
8. 为闪光、震动和顿帧加入强度/无障碍设置滑块。

## 遗留问题 / 待确认

- 生存分 `+2` 是该在每次移动后都弹出，还是仅仅在没有触发任何消除的普通步才弹出？
- 连击乘数是该乘以当次连击步骤中的全体消除得分，还是只乘以**由余震本身引发**的这部分消除得分？（目前的实装是以整个连击深度的层级统一乘的）。
- 混沌模式是否需要比常规模式设置更激进的分数/视觉特效倍率，还是只用更快的动画节奏即可？
- 哪些特效在开始 playtest 前就必须具备强度/无障碍配置滑块？

## 验收标准

- 玩家不看操作日志也能明白为什么分数会增加。
- 高热度的消除必须在主观上比低热度消除显得更有“重量感”。
- 由余震引发的连击，在视觉和数值上都有明显的阶梯式升级。
- 棋盘在爆发连串链式反应期间也能保持清晰可读。
- 所有视觉特效必须通过常量或配置文件调优，不用重新修改玩法核心代码。
