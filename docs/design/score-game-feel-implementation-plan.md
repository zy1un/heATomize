# 计分与打击感 (Game Feel) 实现计划

本计划将当前的计分规则转化为分阶段的实施路径，以实现更强的反馈、更清晰的得分展示以及更具冲击力的连锁反应。

## 设计解读

新的计分方向赋予了 `heATomzie` 独特的个性：得分不再仅仅是数字，它们是热量、电压和被吸入反应堆式计分板的原子碎片。

最稳妥的实现路径是循序渐进的。分层构建得分反馈，在视觉上逐层验证，并避免在节奏未调优之前添加过多同时发生的特效。

最重要的原则：普通移动应当保持清晰和（相对）安静；而连锁反应则应当逐步升级为视觉奇观。

## 核心动画流水线

所有消除事件共享同一条动画管线，连锁时逐级升级。流水线分为 **无倍率路径** 和 **有倍率路径** 两条分支。

### 无倍率路径（chain 1，普通消除）

```
每颗球位置弹出基础分 → 短暂停留 → 数字碎裂为粒子 → 粒子汇集到总分板 → 总分更新
```

1. **球分数弹出**：每颗被消除球的位置弹出 `+N`（基础分 = `10 + 2 × heat`），按温度着色
2. **短暂停留**：约 0.3s，让玩家读数
3. **粒子化汇集**：数字碎裂为温度色的小粒子，飞向总分板区域
4. **总分更新**：辉光管计分板滚动到新分数

### 有倍率路径（chain ≥ 2，连锁消除）

```
每颗球位置弹出基础分 → 倍率横幅弹出 → 顿帧 → 乘后分替换基础分 → 短暂停留 → 全部数字粒子化汇集 → 总分更新
```

1. **球基础分弹出**：每颗球位置弹出 `+N`（基础分），按温度着色——**不粒子化**
2. **倍率横幅**：屏幕中央弹出 `×2` / `×4` / `×8` 横幅，字号随 chain 深度增大
3. **顿帧**（Hit stop）：短暂冻结画面，制造冲击感的"留白"。Phase 5 集中实现
4. **乘后分替换**：基础分被乘后分替换——`+N×M`（如 `+40`），字号略大（设上限阈值），仍按温度着色
5. **短暂停留**：让玩家感知乘后分的变化
6. **粒子化汇集**：所有乘后分数字碎裂为温度色粒子，飞向总分板区域
7. **总分更新**：辉光管计分板滚动到新分数，高分注入时辉光增强

### 乘后分字号阈值

连锁越深，乘后分弹窗越大，但有上限防止数字撑爆格子：

| Chain 深度 | 字号增量 | 说明 |
|-----------|---------|------|
| 2 | +4px | 轻微放大 |
| 3 | +8px | 明显放大 |
| 4 | +12px | 强烈放大 |
| ≥5 | +16px（封顶） | 不再继续增大 |

基础分字号为常量 `PER_BALL_FONT_SIZE`，乘后分字号 = `PER_BALL_FONT_SIZE + min(chain - 1, 4) × 4`。

### 颜色规则

所有分数数字（基础分、乘后分、粒子）严格按照被消除球的热度着色，使用 `HEAT_FILL_COLORS` 和 `HEAT_OUTLINE_COLORS`：
- Heat 1: 冷钢色 (180,196,210)
- Heat 2: 炽红 (230,60,40)
- Heat 3: 烈橙 (255,130,30)
- Heat 4: 熔金 (255,210,50)
- Heat 5: 白热 (255,255,240)

倍率横幅使用固定的金色 `MULTIPLIER_FILL` (255,230,60)。

## 核心特性

1. 每球独立分数弹窗 + 粒子化汇集
2. 辉光管 (Nixie) 风格总分板
3. 连击乘数横幅 + 顿帧 + 乘后分替换
4. 屏幕震动与高能冲击特效
5. 连击动画加速
6. 无障碍 (Accessibility) 友好的特效强度设置

## 实施阶段

### 阶段 1：得分反馈基础 ?

目标：使现有的计分系统可视化，同时不干扰核心玩法。

已完成：
- 在 `board.gd` 中添加 `score_event` 信号
- 创建 `scripts/ui/score_feedback.gd` — 纯视觉 CanvasLayer
- 三种得分事件类型：`survival`、`clear`、`multiplier`
- Press Start 2P 像素字体，按温度着色的弹窗
- 预设菜单整合（MenuButton 替代三个独立按钮）
- 连锁逻辑修复（逐组处理，正确递增 chain）

### 阶段 2：每球独立分数弹窗

目标：将当前的"组总分弹窗"替换为每颗球独立弹出的分数标签。

任务：
- 修改 `score_event` 的 `"clear"` 事件：传递整组信息（所有球的位置、热度、分数），由 score_feedback 为每颗球生成独立标签
- 每颗球在其自身位置弹出基础分 `+N`，按温度着色
- 基础分字号约 24-28px（比组总分小），轻微弹跳动画
- 多球弹窗有微小延迟 stagger（每球间隔 ~0.03s），形成涟漪感
- **移除**当前的组中心总分弹窗
- 连锁倍率横幅保持不变

信号结构设计：
```
score_event.emit("clear", {
    "heat": group_heat,
    "chain_depth": current_chain,
    "multiplier": chain_mult,
    "balls": [{"cell": Vector2i, "heat": int, "base_score": int}, ...],
    "final_score": group_base * chain_mult,
})
```
score_feedback 内部根据 `chain_depth` 决定走无倍率/有倍率路径。

### 阶段 3：辉光管 (Nixie) 计分板

目标：让总分板感觉像一台物理热能设备，同时作为粒子汇集的目标。

任务：
- 将普通的 ScoreLabel 替换为辉光管风格的得分显示器
- 使用飞速滚动的数字制作分数上涨动画（类似老虎机）
- 分数注入时短暂增加辉光效果
- 高乘数时辉光增强，进入"过载"状态
- 提供粒子汇集的终点坐标（ScoreLabel 的屏幕位置）

首版建议：
- 使用带样式的 `Label` 节点 + 主题颜色 + 阴影叠加，模拟橙/黄发光
- 数字快速跳过临时值再停在真实分数

后续版本：
- 自定义数字纹理或发光 Shader
- 高乘数时火花粒子环绕计分板

### 阶段 4：原子化分数粒子

目标：使分数在物理上表现出从被消除的球飞向计分板的过程。

任务：
- 分数数字短暂悬停后，碎裂为温度色的小方形/圆形粒子
- 粒子飞向辉光管计分板区域（阶段 3 提供的坐标）
- 粒子颜色严格与被消除球的热度颜色一致
- 粒子到达计分板后消失，触发计分板滚动更新

实施注意事项：
- 复用当前像素碎片爆炸的视觉语言（`spawn_transfer_pixel` 模式）
- 单球粒子数量控制在 4-6 个，大规模消除时总粒子数不超过 ~50
- 粒子飞行时间约 0.4-0.6s，曲线轨迹（抛物线或贝塞尔）而非直线
- 到达计分板时有微小的"吸入"缩放动画

### 阶段 5：顿帧与倍率冲击

目标：让连锁消除（chain ≥ 2）的得分过程充满戏剧性。

任务：
- **顿帧辅助函数**：冻结游戏逻辑指定帧数，视觉画面静止但不卡死
- **乘后分替换**：基础分标签在顿帧结束后被乘后分替换（字号放大，按温度着色）
- **倍率横幅升级**：
  - `×2`：中等横幅，极短顿帧（2-3 帧）
  - `×4`：更大横幅，更强发光，顿帧 3-4 帧
  - `×8+`：过载表现，屏幕震动，四周爆出火花，顿帧 4-6 帧
- **屏幕震动辅助函数**：在视口内偏移 board 节点位置
- Game Over 时的停机顿帧及画面逐渐暗淡的谢幕序列

实施警告：
- 所有震动都在游戏视口内进行，不操作物理窗口
- 顿帧期间确保 `process` 暂停但渲染不中断

### 阶段 6：连击加速

目标：初期消除保持可读性，连击不断加速，演变成视觉奇观。

设计：
- Chain 1：正常速度 `1.0x`
- Chain 2：约 `1.25x`
- Chain 3：约 `1.5x`
- Chain ≥4：极限 `2.0x` ~ `2.5x`

节奏乘数作用于：
- 热量传导粒子持续时间
- 阶段暂停时间
- 消除收缩动画时间
- 余震传递动画时间
- 得分粒子飞行时间
- 基础分/乘后分停留时间

节奏乘数不作用于：
- 玩家的移动动画
- 新球生成展示
- 底层规则计算

### 阶段 7：视觉测试

目标：阻止花哨的特效意外遮挡或破坏游戏局势的可读性。

任务：
- 针对以下场景添加基于截图的冒烟测试：
  - 正常棋盘加载后状态
  - 规则面板可见性
  - 计分板可见性
  - 被选中的球可见性
  - 消除反馈场景
  - 超高乘数场景
- 确定性的调试棋盘用于自动化测试
- 验证单球弹窗在 7 球大规模消除时不遮挡关键信息

## 验收标准

- 玩家不看操作日志也能明白为什么分数会增加。
- 高热度的消除必须在主观上比低热度消除显得更有"重量感"。
- 由余震引发的连击，在视觉和数值上都有明显的阶梯式升级（基础分 → 倍率 → 乘后分）。
- 棋盘在爆发连串链式反应期间也能保持清晰可读。
- 所有视觉特效必须通过常量或配置文件调优，不用重新修改玩法核心代码。
- 粒子汇集让"分数飞向计分板"有物理感，不是数字凭空出现。
